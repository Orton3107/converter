# Инструкции по настройке Supabase для Конвертера Валют

Этот документ описывает шаги для интеграции Supabase в ваше PWA-приложение "Конвертер Валют".

## 1. Настройка проекта в Supabase

### 1.1. Создание проекта
1. Зарегистрируйтесь на [https://supabase.com/](https://supabase.com/)
2. Создайте новый проект.

### 1.2. Получение ключей
После создания проекта найдите:
- **URL проекта**: Находится в настройках проекта (`Project Settings` > `API`).
- **anon ключ**: `Project Settings` > `API` > `Project keys` > `anon` public.
- **service_role ключ**: `Project Settings` > `API` > `Project keys` > `service_role` (только для серверного использования, не передается в фронтенд).

### 1.3. SQL для создания таблицы и политик RLS
В SQL Editor вашего Supabase проекта выполните следующий код:

```sql
-- Создаем таблицу для хранения курсов валют
CREATE TABLE currency_rates (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    currency_code VARCHAR(3) NOT NULL UNIQUE, -- Код валюты (USD, EUR, UAH, MDL)
    rate_to_usd DECIMAL(18, 8) NOT NULL,      -- Курс относительно 1 USD
    last_updated TIMESTAMPTZ DEFAULT now()    -- Время последнего обновления
);

-- Создаем политику безопасности для публичного доступа на чтение
CREATE POLICY "Public can view all currency rates" ON currency_rates
    FOR SELECT USING (true);

-- Создаем политику безопасности для сервисной роли на изменение данных
CREATE POLICY "Service role can manage currency rates" ON currency_rates
    FOR ALL USING (
        auth.role() = 'service_role'
    );

-- Включаем RLS для таблицы
ALTER TABLE currency_rates ENABLE ROW LEVEL SECURITY;

-- (Опционально, но рекомендуется) Создаем функцию для обновления времени
CREATE OR REPLACE FUNCTION update_last_updated_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.last_updated = now();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- (Опционально, но рекомендуется) Создаем триггер для автоматического обновления времени
CREATE TRIGGER update_currency_rates_last_updated
    BEFORE UPDATE ON currency_rates
    FOR EACH ROW
    EXECUTE FUNCTION update_last_updated_column();
```

## 2. Настройка фронтенда (PWA)

### 2.1. Обновление HTML файла
Убедитесь, что в `index.html` в секции `<head>` добавлен скрипт Supabase SDK:
```html
<!-- Supabase SDK для фронтенда -->
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
```

### 2.2. Конфигурация Supabase в JavaScript
Откройте файл `app.js` и замените заглушки на реальные значения:
```javascript
// ВАЖНО: Замените эти значения на ваши реальные ключи из настроек Supabase проекта
const SUPABASE_URL = 'YOUR_SUPABASE_URL_HERE'; // e.g., 'https://xyzabcd.supabase.co'
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY_HERE'; // Ваш публичный anon ключ
```

### 2.3. Логика работы с данными
В `app.js` реализована функция `fetchRates()`, которая:
1. Сначала пытается загрузить курсы из таблицы `currency_rates` в Supabase.
2. Если данные в Supabase отсутствуют, использует локальный кэш.
3. Если и кэш пуст, можно добавить логику fallback к внешнему API (опционально).

Локальный кэш (`localStorage`) используется для работы в офлайн-режиме и ускорения загрузки.

## 3. Настройка серверной части (Node.js скрипт)

### 3.1. Установка зависимостей
Убедитесь, что вы выполнили команду в корне вашего проекта:
```bash
npm install
```
Это установит пакет `@supabase/supabase-js`.

### 3.2. Конфигурация скрипта обновления
Откройте файл `update-rates.js` и замените заглушки на реальные значения:
```javascript
// ВАЖНО: Замените эти значения на ваши реальные ключи из настроек Supabase проекта
const SUPABASE_URL = 'YOUR_SUPABASE_URL_HERE'; // e.g., 'https://xyzabcd.supabase.co'
const SUPABASE_SERVICE_ROLE_KEY = 'YOUR_SUPABASE_SERVICE_ROLE_KEY_HERE'; // Ваш сервисный ключ
```

### 3.3. Запуск скрипта
Для обновления курсов валют выполните команду в терминале (в корне проекта):
```bash
npm run update-rates
```
Скрипт:
1. Получит актуальные курсы из внешнего API (`https://open.er-api.com/v6/latest/USD`).
2. Обновит или вставит данные в таблицу `currency_rates` в Supabase, используя `service_role_key` для обхода RLS.

## 4. Автоматизация обновления (опционально)

Вы можете настроить автоматический запуск скрипта `update-rates.js` (например, через cron-задачи на вашем Mac mini) для регулярного обновления курсов. Например, раз в день.

## 5. Важные замечания

- **Безопасность**: Никогда не публикуйте `service_role_key` в публичном доступе (например, в исходном коде фронтенда или в репозитории GitHub). Он используется только в доверенных скриптах на вашем сервере или локальной машине.
- **Обработка ошибок**: В текущей реализации, если Supabase недоступен, приложение должно корректно переключиться на использование локального кэша.
- **Тестирование**: После настройки тщательно протестируйте приложение в разных условиях (онлайн/офлайн, при отсутствии данных в Supabase и т.д.).

## 6. Структура проекта после изменений

- `app.js`: Основной логик PWA, теперь использует Supabase для получения данных.
- `update-rates.js`: Node.js скрипт для синхронизации данных с Supabase.
- `package.json`: Управляет зависимостями, включая `@supabase/supabase-js`.
- `index.html`: Обновлен для включения Supabase SDK.

## 7. Устранение неполадок

- **"Ошибка Supabase:..."**: Проверьте правильность URL и `anon_key` в `app.js`. Убедитесь, что проект в Supabase активен.
- **"Не удалось загрузить курсы"**: Проверьте соединение с интернетом. Убедитесь, что данные есть в Supabase или в локальном кэше.
- **Скрипт `update-rates.js` не работает**: Проверьте наличие Node.js, правильность `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY` в скрипте, а также выполнение `npm install`.
